# 溫控通知等級

!!! note "已棄用功能"
    此功能已被 iOS 棄用，在 iOS 16 或更高版本中不再可用。

Battman 會展示系統的**溫控通知等級**，用於描述 iOS 在發熱時會採取多激進的限制措施。

## 等級

熱通知系統使用一系列漸進的行為等級，每個等級代表越來越激進的熱緩解措施。下方顯示的數字是**行為索引**（0-10），而非實際的系統通知等級 ID。

0. **正常** — 正常工作狀態，無熱限制。

1. **閃光燈衰減至70%** — 手電筒（閃光燈）亮度降至 70%。

2. **背光衰減至70%** — 顯示屏背光降至 70%。

3. **閃光燈衰減至50%** — 手電筒亮度進一步降至 50%。

4. **背光衰減至50%** — 顯示屏背光降至 50%。

5. **閃光燈已禁用** — 手電筒完全禁用。

6. **背光衰減至25%** — 激進的背光降低至 25%。

7. **地圖指南針已禁用** — 禁用地圖指南針等視覺效果。

8. **App 終止運行** — 可能會終止應用程序以降低熱負載。

9. **設備重新啓動** — 極端熱條件可能觸發設備重啓。

10. **已準備好** — 熱表已準備好（內部狀態）。

## 實現細節

**重要：** 實際的系統通知等級 ID 由 iOS**動態分配**，並非固定值。上方顯示的行為索引（0-10）只是不同熱緩解行為的標識符。

這種動態映射使 Battman 即使在 iOS 為相同的熱行為分配不同數字 ID 時也能正常工作。

## 在 Battman 中

- **溫控調校** 頁面會展示當前的通知等級，並允許有限的覆蓋。
- **硬件溫度** 頁面可幫助你將這些等級與具體傳感器讀數對應起來。

如果設備經常處於較高的通知等級，建議降低負載、改善散熱，或在溫控調校中採用更保守的設置。
